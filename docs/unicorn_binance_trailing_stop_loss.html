class unicorn_binance_trailing_stop_loss.manager.BinanceTrailingStopLossManager(binance_public_key: Optional[str] = None, binance_private_key: Optional[str] = None, callback_error: Optional[builtin_function_or_method] = None, callback_finished: Optional[builtin_function_or_method] = None, exchange: str = 'binance.com', keep_threshold: Optional[str] = None, print_notificatons: bool = False, reset_stop_loss_price: bool = False, send_to_email_address: Optional[str] = None, send_from_email_address: Optional[str] = None, send_from_email_password: Optional[str] = None, send_from_email_server: Optional[str] = None, send_from_email_port: Optional[int] = None, start_engine: bool = True, stop_loss_limit: Optional[str] = None, stop_loss_market: Optional[str] = None, stop_loss_order_type: Optional[str] = None, stop_loss_price: Optional[float] = None, stop_loss_side: Optional[str] = None, stop_loss_trigger_gap: str = '0.01', telegram_bot_token: Optional[str] = None, telegram_send_to: Optional[str] = None, test: Optional[str] = None, trading_fee_discount_futures_percent: float = 10.0, trading_fee_discount_margin_percent: float = 25.0, trading_fee_discount_spot_percent: float = 25.0, trading_fee_percent: float = 0.1, trading_fee_use_bnb: bool = False, warn_on_update=True)¶

Bases: threading.Thread

unicorn-binance-trailing-stop-loss for managing Stop/Loss orders and sending notifications.

Supported exchanges: binance.com-isolated_margin

Parameters

  • binance_public_key (str) – Provide the public Binance key.

  • binance_private_key (str) – Provide the private Binance key.

  • callback_error (function) – Callback function used if an error occurs.

  • callback_finished (function) – Callback function used if stop_loss gets filled.

  • exchange (str) – Choose the exchange endpoint: binance.com, binance.com-futures, binance.com-margin, binance.com-isolated_margin

  • keep_threshold (str) – If empty we sell the full balance, use integer or percent values.

  • print_notificatons – If True the lib is printing user friendly information to terminal.

  • reset_stop_loss_price (bool) – Reset an existing stop_loss_price and calculate a new one. Only True is True, anything else is False!

  • stop_loss_limit (str) – The limit is used to calculate the stop_loss_price of the highest given price, use integer or percent values.

  • send_to_email_address (str) – Email address of receiver

  • send_from_email_address (str) – Email address of sender

  • send_from_email_password (str) – Password for SMTP auth

  • send_from_email_server (str) – Hostname or IP of SMTP server

  • send_from_email_port (int) – Port of SMTP server

  • start_engine (bool) – Start the trailing stop loss engine. Default is True

  • stop_loss_market (str) – The market to enforce stop/loss.

  • stop_loss_order_type (str) – Can be limit or market - default is None which leads to a stop of the algorithm.

  • stop_loss_price (float) – Set a price to use for stop/loss, this is valid till it get overwritten with a higher price.

  • stop_loss_side (str) – Can be buy or sell - default is None which leads to a stop of the algorithm.

  • stop_loss_trigger_gap (str) – Gap between stopPrice and limit order price, use integer or percent values.

  • test (str) – Use this to test specific systems like “notification”. If test is not None the engine will NOT start! It only tests!

  • telegram_bot_token (str) – Token to connect with Telegram API.

  • telegram_send_to (str) – Receiver of the message sent via Telegram.

  • trading_fee_use_bnb (bool) – Default is False. Set to True to use BNB for a discount on trading fees: https://www.binance.com/en/support/faq/115000583311.

  • warn_on_update (bool) – set to False to disable the update warning
